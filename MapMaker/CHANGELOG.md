# Map Maker - Changelog

All notable changes to the Super Battle Golf Map Maker will be documented in this file.

## [1.0.0] - 2026-04-14

### 🎉 Initial Release

Complete Unity-based map creation tool for Super Battle Golf custom courses.

### Added

**Core Features:**
- Unity 6000.3.10f1 project template
- Addressables 2.8.1 integration
- Automated build pipeline
- One-click export to mod folder format

**Editor Tools:**
- **MapMakerWindow** - 3-step workflow (Setup → Build → Export)
  - Auto-generates mod.json from GUI fields
  - Validates Addressables naming (CustomCourse prefix)
  - Builds asset bundles with one click
  - Exports ready-to-install mod folder
  - Desktop output option for easy distribution

**Components:**
- **CourseData** - Course-level metadata
  - Name, description, author, version
  - Difficulty, environment, tags
  - Total holes calculation
  - Lighting and audio settings
  
- **HoleData** - Per-hole configuration
  - Hole number, par, name
  - Tee/cup transform references
  - Playable bounds visualization
  - Hazard arrays (water, sand, OOB)
  - Visual gizmos (green=tee, red=cup, yellow=bounds)

**GameObject Menu:**
- **Course Elements**
  - Tee Box (platform + markers)
  - Cup (flag + flagpole + hole)
  
- **Hazards**
  - Water Hazard (trigger zone)
  - Sand Trap (rough terrain)
  
- **Obstacles**
  - Tree (tall cylinder)
  - Rock (sphere)
  
- **Create Complete Hole** (tee + fairway + cup in one click)

**Factory Methods:**
- **CourseElementFactory** static class
  - CreateTeeBox() - Blue/red markers
  - CreateCup() - Flag and hole
  - CreateWaterHazard() - Blue trigger
  - CreateSandTrap() - Yellow collider
  - CreateObstacle() - Generic cylinder

**Scene Utilities:**
- **SceneInitializer** - Auto-generates example scene
  - Creates 3 complete holes
  - Adds lighting and ground plane
  - Pre-configured CourseData
  - Ready to build immediately

**Documentation:**
- **README.md** (8KB) - Complete guide
  - Quick start workflow
  - Component reference
  - Step-by-step tutorial
  - Troubleshooting
  - Publishing guide
  
- **QUICKSTART.md** (3.4KB) - 5/15-minute guides
  - Fastest path to working map
  - Pro tips
  - Common mistakes
  - What's next

### Technical Details

**Unity Configuration:**
- Unity 6000.3.10f1 (exact version match required)
- Addressables 2.8.1
- URP 17.0.3
- Build target: Windows (Standalone)
- Scripting backend: Mono
- API compatibility: .NET Framework

**File Structure:**
```
MapMaker/
├── Assets/
│   ├── Scenes/              (empty, user creates)
│   ├── Scripts/
│   │   ├── Editor/
│   │   │   ├── MapMakerWindow.cs      (10KB)
│   │   │   ├── CourseElementMenu.cs   (5.9KB)
│   │   │   └── SceneInitializer.cs    (8.7KB)
│   │   ├── HoleData.cs                (2.7KB)
│   │   ├── CourseData.cs              (1.7KB)
│   │   └── CourseElementFactory.cs    (5.9KB)
│   └── AddressableAssetsData/
│       └── (generated by Addressables)
├── Packages/
│   └── manifest.json        (Addressables dependency)
├── ProjectSettings/
│   └── (Unity configuration)
├── README.md                (8KB)
└── QUICKSTART.md            (3.4KB)
```

**Workflow:**
1. **Setup** - Configure Addressables with correct naming
2. **Build** - Compile asset bundles for Windows
3. **Export** - Generate mod.json + copy bundles to output folder

**Output Format:**
```
MyMapName/
├── mod.json              (auto-generated)
├── README.md             (installation instructions)
├── catalog.bin           (Addressables catalog)
└── [bundle files]        (asset data)
```

### Design Decisions

**Why Unity 6000.3.10f1?**
- Exact version match with Super Battle Golf
- Asset bundles must be built with same Unity version
- Binary compatibility requirement

**Why Addressables?**
- Game already uses Addressables for map loading
- Native Unity system (no third-party tools)
- Supports async loading and dependency management
- Proven to work with mod loader

**Why Desktop export?**
- Easy sharing (drag/drop folder to Discord, etc.)
- One location for all exported maps
- No risk of accidentally putting in game folder during dev

**Why separate components?**
- CourseData = whole course (shared data)
- HoleData = individual holes (reusable, modular)
- Clean separation of concerns
- Easy to clone/duplicate holes

### Known Limitations

**Current Version:**
- No multi-scene support (planned for v1.1)
- No prefab library (planned for v1.2)
- Manual terrain creation (could add templates)
- Windows-only build (Mac/Linux not tested)

**Unity Editor Required:**
- This is an editor tool, not a standalone app
- Users must install Unity 6000.3.10f1
- Could create standalone builder in future
